Phase 1 JavaScript & TypeScript Foundations ~2–3 weeks ▶
Start here — Angular is written in TypeScript. Think of it as C# for the browser.
JS Basics
Variables (let/const), arrays, objects, loops, functions
like C# basics
ES6+ Features
Arrow functions, destructuring, spread, template literals, modules
like LINQ & lambda
Async JS
Promises, async/await, callbacks, fetch API
like Task/await in C#
TypeScript Core
Types, interfaces, classes, generics, enums, decorators
very similar to C#
TypeScript Advanced
Union types, type guards, utility types (Partial, Readonly, Pick)
DOM Basics
How browsers render HTML, event handling, the document tree
📚 Best resources: JavaScript.info (free, excellent), TypeScript Handbook at typescriptlang.org
Phase 2 Angular Core Concepts ~3–4 weeks ▶
The building blocks of every Angular app. This is where it clicks for .NET devs.
Angular CLI
ng new, ng serve, ng generate — like dotnet CLI
like dotnet CLI
Components
Creating, structuring, and nesting components with @Component decorator
like ASP.NET views + controllers
Templates & Directives
ngIf, ngFor, ngSwitch, ngClass, ngStyle, template syntax
Data Binding
One-way [property], event (event), two-way [(ngModel)], interpolation {{ }}
Modules (NgModules)
AppModule, feature modules, imports, declarations, exports
like C# namespaces/assemblies
Component Lifecycle
ngOnInit, ngOnChanges, ngOnDestroy, and when to use each
Input & Output
@Input() and @Output() for parent-child communication
Pipes
Built-in pipes (date, currency, uppercase) and creating custom pipes
📚 Best resource: Official Angular Tour of Heroes tutorial at angular.io — do it top to bottom
Phase 3 Services, DI & HTTP ~2–3 weeks ▶
This is where your .NET background gives you a huge advantage — DI is nearly identical.
Dependency Injection
providedIn root/module, injector hierarchy, DI tokens
nearly identical to .NET DI
Services
@Injectable, creating services, singleton vs scoped instances
like .NET services / repositories
HttpClient
GET, POST, PUT, DELETE calls to REST APIs, request/response typing
like HttpClient in .NET
RxJS Basics
Observables, subscribe(), pipe(), map, filter, switchMap
like IObservable / LINQ + async
Error Handling
catchError, throwError, HTTP interceptors for global error handling
Interceptors
HTTP interceptors for auth headers, logging, retries
like ASP.NET middleware
📚 Best resource: RxJS.dev guide + Fireship's "RxJS Top 10 Operators" on YouTube
Phase 4 Routing, Forms & State ~2–3 weeks ▶
Build real apps that navigate between pages and handle complex forms.
Angular Router
RouterModule, routes config, routerLink, ActivatedRoute, query params
Route Guards
CanActivate, CanDeactivate, Resolve — protect routes
like ASP.NET authorization filters
Lazy Loading
Load feature modules on demand to improve performance
Template-Driven Forms
FormsModule, ngModel, simple validation
Reactive Forms
FormBuilder, FormGroup, FormControl, Validators — more powerful
like FluentValidation in .NET
State Management
Services as state, BehaviorSubject, intro to NgRx (optional)
📚 Reactive Forms > Template Forms for .NET devs — the explicit control feels familiar. Start there.
Phase 5 Advanced & Production-Ready ~3–4 weeks ▶
Level up to build scalable, production-quality Angular applications.
Change Detection
Default vs OnPush strategy, ChangeDetectorRef, performance
Standalone Components
Angular 17+ modern approach — no NgModules required
Unit Testing
Jasmine + Karma, TestBed, mocking services, component testing
like xUnit/NUnit in .NET
NgRx (State)
Actions, reducers, effects, selectors — Redux pattern for Angular
Performance
trackBy in ngFor, async pipe, virtual scrolling, bundle optimization
Angular + .NET Integration
Connecting your Angular app to ASP.NET Core Web API — your sweet spot
your biggest advantage!
📚 Best resource: angular.io/guide (official docs are excellent) + ultimatecourses.com by Todd Motto
💡 Your .NET superpower: DI, strong typing, services, and REST APIs will feel very natural. RxJS is the biggest new concept — give it extra time.
